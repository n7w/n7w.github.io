kmp与状态机

发表于 2020-02-24 分类于 算法

引言

上VHDL，遇到一道题序列检测器：检测到连续信号为"1110010"时输出1，突然联想到一直没搞懂的KMP算法：在主文本字符串S内查找一个词W出现的位置。借此机会好好学了一下，然后试图把状态机的想法融入KMP中

原题

出自《VHDL数字系统设计》5.6 设计实例1————序列检测器 p113
由ASM图可以看出，当检测到111X时，如果X=1，则回退到111X的状态，并不是回归初态。其他点同上。

KMP

KMP也是这么操作的，详见阮一峰老师的KMP算法讲解

根据前缀和后缀得出的部分匹配表，结合ASM图
若次态失配，参照现态对应的部分匹配值i，将现态回退至目标序列的第i个状态
如本题中部分失配表

搜索词 1 1 1 0 0 1 0
部分匹配值 0 1 2 0 0 1 0

当检测到111时，若次态不为0，根据现态与部分匹配值，将现态回退到第2个状态，也就是11

这时如果你对着书看你就会质问：不对啊，明明书上是回退到111啊，你这不对啊！
我一开始也挺疑惑，后来说服了自己：KMP算法是针对任意字符串的匹配，而实际信号只有1和0，不用考虑其他因素，故可在KMP算法基础上补充，然后就有了书上更优的ASM图

部分失配表

问题来了，如何用算法生成这个表呢？
在比较多程序算法中，由于我们往往是判断现态是否匹配（即S[m] == W[i]），而失配表是判断次态的（即S[m+1] == W[i]），因此我们选择把匹配值右移，把第一位填充为-1，即

i 0 1 2 3 4 5 6
W 1 1 1 0 0 1 0
T -1 0 1 2 0 0 1

得到的T[i]为现态W[i]失配时要回退到第T[i]个状态
一般把这个T[]记作next[]

对任意字符串S，我们容易知道，T[0] = -1; T[1] = 0;
而对于其他匹配值，实质上是子字符串W[:i]（即[0,i)）的最长的共有元素的长度。（因为右移）

以下为Wiki上KMP的数学伪代码

由代码可见，cnd就是最长的共有元素的长度，也就是失配值
从逻辑上我们可以知道，之所以有KMP算法，是因为字符串里可能有AB..........AB........，这样做就能减少回溯
first case里的条件成立时，pos和cnd都指向下一个状态，同时cnd记录着现在最大的共有元素长度
故在O(n)内可建立部分失配表

面向对象编程原则

发表于 2021-02-02 更新于 2021-02-13

面向对象编程原则

支持可维护性复用，一方面需要实现设计方案或代码的复用，另一方面要保证系统易于扩展和修改，具有良好的可维护性。

单一职责原则

一个类应只负责一种职责，比如逻辑和display应该分离

如何分离

类A有职责T1和T2，T1需求更新导致A类要重写，若重写导致T2出BUG，则说明T1和T2是耦合的，应当分离

开放封闭原则

对扩展性开放，对更改是封闭的
即尽可能不修改类，而是在父类的基础上扩展出子类

里氏替换原则

软件使用基类时，可以用任一个子类来替代父类
即Base A = new AnyDerived()

继承的要求

  1. 子类不可重写父类方法（Override）
  2. 子类可重载父类方法（Overload），此时输入范围>=父类，输出范围`<=``父类
  3. 子类可以扩展（新增方法）

依赖倒置原则

  • 高层模块不应该依赖低层模块，二者都应该依赖其抽象
  • 抽象不应该依赖细节，细节应该依赖抽象

例子：妈妈读书

高层模块直接依赖于低层模块，如果此时高层模块增加新的需求，即妈妈可以阅读报纸 class Paper ，则需要修改 class Mother 的代码

此时应该设置抽象层

迪米特法则

如果两个类不必彼此直接通信，那么这两个类就不应当发生直接的相互作用。如果其中的一个类需要调用另一个类的某一个方法的话，可以通过第三者转发这个调用。

接口隔离原则

  • 客户端不应该依赖它不需要的接口
  • 一个类对另一个类的依赖应该建立在最小的接口上（细分大型接口）

合成复用原则

在软件设计时，优先使用关联、聚合和组合关系，尽量少用泛化（继承）
如果基类发生改变，那么子类的实现也不得不改变。

design-mode-prototype

发表于 2021-02-05 更新于 2021-02-06

原型模式

原型模式要求对象实现一个可以克隆自身的接口(类型)。这样一来，通过原型实例创建新的对象，就不需要关心这个实例本身的类型，只需要实现克隆自身的方法，也而无需再去通过new来创建。

浅拷贝

参考 Java深拷贝和浅拷贝

实现

浅拷贝的浅

如果属性是基本类型，拷贝的就是基本类型的值
如果属性是内存地址（引用类型），拷贝的就是内存地址

  1. String 为 final，故只会修改引用指针，student2.name 指向新的地址 "alice"
  2. Integer 是基本类型，会拷贝出新的地址，student2.age 指向新的地址 20
  3. student2.subject 指针指向新的地址 Subject("english")
  4. student2.subject.name 修改了指向，但 student2.subject 指向的地址不变，和 student1.subject 指向同一份地址，故两者的 subject 都变了

第四点为浅拷贝的浅

深拷贝

深拷贝会拷贝所有的属性，并拷贝属性指向的动态分配的内存

手动深拷贝

对象简单时直接对每个参数 new 一遍即可，对象复杂时使用 DFS 进行深拷贝

design-mode-singleton

发表于 2021-02-03 更新于 2021-02-05

单例模式

确保一个类有且只有一个实例，并提供一个全局访问点。

在开发中，很多时候有一些对象其实我们只需要一个，例如：线程池（threadpool）、缓存（cache）、默认设置、注册表（registry）、日志对象等等，这个时候把它设计为单例模式是最好的选择。

饿汉型

初始化即装载单例，由于放在静态区内，100%线程安全

懒汉型（线程不安全）

使用时才装载单例（延迟加载）

懒汉型（线程安全）

将整个函数加锁

懒汉型（双重校验锁式，线程安全）

大多数情况下，单例已经生成了，但整个函数是加锁的，意味着即使已经生成了单例但在获取的时候还是加了锁，应对此进行优化

静态内部类单例

单例模式（Singleton Pattern）之静态内部类

枚举单例

Java单例模式的7种写法中，为何用Enum枚举实现被认为是最好的方式？

枚举类本身就是一种单例，只需要将单例类改成枚举类即可

LRU 和 LFU

发表于 2020-09-07 更新于 2020-09-09

Least Recently Used (LRU)

最近最久使用算法
满足查询 $O(1)$，插入 $O(1)$，删除 $O(1)$
实现方式：LinkedHashMap，即双向链表 + 哈希表

Least Frequently Used (LFU)

最近最少使用算法
满足查询 $O(1)$，插入 $O(1)$，删除 $O(1)$
实现方式：DLinkList<DLinkList>，即以双向链表为节点的双向链表
